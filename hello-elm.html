  • Fantastic Error Messages

Basic Elm Syntax

  • Functions
  • Basic Data Structures
  • Functional Transforms
  • Elm Html Library

Functions


add a b =
    a + b

add 2 3
=> 5
  • Functions must return exactly one value
  • Functions have no side-effects

Functions


squareOfSum a b =
  let
      sum = a + b
  in
      sum * sum  

squareofSum 2 3
=> 25
  • Functions can contain local, intermediate functions

Functions


lambda =
    \a b -> a + b

lambda 3 5
=> 8
  • This is an anonymous function (lambda)
  • Functions can consume and return other functions

Lists


myList = [2, 3,4]
=> [2, 3, 4]

newList = 1 :: myList
=> [1, 2, 3, 4]

myList ++ newList
=> [1, 2, 3, 4, 2, 3, 4]

Records


type alias Person =
    { name : String
    , age : Int
    }

p1 = Person "James" 23
p2 = { name = "Jane", age = 25 }

p1.name 
=> "James"

p3 = { p2 | age = 35 }
=>  { name = "Jane", age = 35 }

Functional Transforms


numbers = [1, 2, 3]

List.map (\elem -> elem * elem) numbers
=> [1, 4, 9]

List.filter (\elem -> elem > 2) numbers
=> [3]

numbers |> List.map (\elem -> elem * elem)
=> [1, 4, 9]

Elm Html Lib

  • Every Html element has a corresponding function
  • Each function takes two paramaters
    • First param is a list of Html attributes
    • Second param is a list of Html sub-elements
  • Function nesting can be arbitrarily deep

Examples


h1 [ class "alert"] [ text "Alert Text" ]

div [ class "container", id "main-content" ] 
    [ p [] [ text "paragraph 1" ]
    , p [] [ text "paragraph 2" ]
    ]

Creating a Data Model


type alias TodoItem =
    { id : Int
    , description : String
    , completed : Bool
    }


type alias Model =
    { nextId : Int
    , items : List TodoItem
    }

Creating Seed Data


item2 =
    TodoItem 2 "Clean litter box" False


initModel =
    [ TodoItem 1 "Take out trash" False
    , item2
    , TodoItem 3 "Finish meetup talk" True
    ]

Display a single Todo


singleItem item =
    let
        singleCheckBox item =
            input
                [ style [ ( "margin-right", "8px" ) ]
                , type_ "checkbox"
                , checked item.completed
                ]
                []
    in
        div [ class "alert alert-success" ]
            [ singleCheckBox item
            , text item.description
            ]

Add content to our page


content =
    div [ class "col-md-9" ] [ singleItem item2 ]

Render all Todos


content model =
    div [ class "col-md-9" ]
        (List.map
            (\item -> singleItem item)
            model.items
        )

Pass Model as Param


main =
    div []
        [ header
        , mainBody initModel
        ]


mainBody model =
    div
        [ class "row container" ]
        [ sidebar
        , content model
        ]

Now Let's Make It Do Something

Elm Architecture

Elm Apps Use a Common Architecture

  • Model - All Data for the App
  • View - Builds HTML from data in Model
  • Update - A Function that Updates a Model

Tying it Together


main =
    Html.beginnerProgram
        { model = initModel
        , update = update
        , view = view
        }

Tying it Together


update msg model =
    model

view model =
    div []
        [ header
        , mainBody initModel
        ]
